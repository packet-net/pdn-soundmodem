# Figure D-9 — Constraint length 7, rate 1/2 convolutional encoder

Source: MIL-STD-188-110D Appendix D, D.5.3.2.1 + Figure D-9, document page 219
(PDF page 224). Image page (no text layer); transcribed by eye at 300 dpi, tap
arrows traced on a 2x-upscaled crop.

## Polynomials as printed (heading "Polynomials:")

```
(b0) T1 = x^6 + x^4 + x^3 + x^1 + 1
(b1) T2 = x^6 + x^5 + x^4 + x^3 + 1
```

## Prose (D.5.3.2.1, transcribed from image)

> A constraint length 7, rate 1/2 convolutional code shall be used prior to
> puncturing. This shall be the same code as is used in the single-tone waveform
> described in section 5.3.2 of this standard. Figure D-9 is a pictorial
> representation of the encoder.

Below the figure:

> The two summing nodes in the figure represent modulo 2 additions. For each bit
> input to the encoder, two bits are taken from the encoder, with the upper
> output bit b0, generated by polynomial T1(x), taken first.

## Figure wiring, tap by tap

- Shift register of 7 cells, drawn left to right and labeled
  `X^6, X^5, X^4, X^3, X^2, X, 1`.
- The **Input arrow enters the leftmost cell (`X^6`)**; bits shift rightward,
  so `X^6` holds the newest input bit and `1` holds the oldest (six shifts ago).
- **Upper adder → b0 (T1)**: taps from cells `X^6, X^4, X^3, X, 1`
  (five arrows; matches the printed T1 exactly).
- **Lower adder → b1 (T2)**: taps from cells `X^6, X^5, X^4, X^3, 1`
  (five arrows; matches the printed T2 exactly).
- Both adders are modulo-2. Output order per input bit: **b0 first, then b1**.

## Codable form (delay-operator convention)

With u[n] the current input bit (cell `X^6`) and u[n-6] the oldest (cell `1`):

```
b0[n] = u[n] ^ u[n-2] ^ u[n-3] ^ u[n-5] ^ u[n-6]   ->  1 + D^2 + D^3 + D^5 + D^6  =  1011011b  =  0o133
b1[n] = u[n] ^ u[n-1] ^ u[n-2] ^ u[n-3] ^ u[n-6]   ->  1 + D + D^2 + D^3 + D^6    =  1111001b  =  0o171
```

i.e. the standard (133, 171) octal K=7 pair, T1 = 133 emitted first. (The
printed polynomial letters x^k index register cells with x^6 = newest, so the
printed form is the bit-reversed spelling of the same taps; the figure wiring
above is the ground truth.)

Note: the same page's prose ties this to main-body 5.3.2 ("same code as ...
section 5.3.2"), which is the published 133/171 code — consistent.
