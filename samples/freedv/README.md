# FreeDV datac reference vectors

Checked-in codec2 reference transmissions for the OFDM demodulator tests
(`tests/.../Ofdm/DatacDemodTests.cs`). They let the suite run without a
libcodec2 build.

## Provenance

Generated by libcodec2 **1.2.0, git `310777b`** (LGPL-2.1) via a small harness
(`tools/gen_datac0.c`, not committed — it is a test oracle, kept with the codec2
checkout) that calls `freedv_rawdatacomptx(FREEDV_MODE_DATAC0, …)` repeatedly to
emit a **continuous stream of identical datac0 modem packets** (no preamble /
postamble), i.e. the streaming path that `freedv_open(FREEDV_MODE_DATAC0)` uses.
codec2's own streaming RX (`freedv_open` + `freedv_rawdatarx`, no
`freedv_set_frames_per_burst`) decodes every one of these; our pure-managed
demodulator must match.

Each `.s16` is raw signed-16-bit little-endian mono at **8000 Hz**. The frame the
demodulator must recover is the same for every vector and is stored in
`datac0_clean.frame` (16 bytes = 14-byte payload `05 16 27 … e2` + big-endian
CRC-16 `24 b1`, the CRC codec2 appends).

| file | packets | carrier offset | AWGN σ (int16) |
|------|---------|----------------|----------------|
| `datac0_clean.s16`      | 10 | 0 Hz  | 0    |
| `datac0_foff.s16`       | 10 | +45 Hz | 0   |
| `datac0_noise.s16`      | 20 | 0 Hz  | 6000 |
| `datac0_noise_foff.s16` | 20 | +35 Hz | 3000 |

Signal RMS ≈ 7400; the datac0 signal occupies ~500 Hz of the 4 kHz band, so the
in-band SNR is well above the wideband figure — the LDPC recovers 19/20 at
σ=6000. The sample-clock-offset case is produced at test time by resampling
`datac0_clean.s16` (±600 ppm), no separate fixture.

## Regenerating

```sh
cd <codec2-1.2.0 checkout>
cmake -B build -DCMAKE_BUILD_TYPE=Release && cmake --build build -j1
gcc -O2 -o gen_datac0 gen_datac0.c -Isrc -Ibuild -Lbuild/src -lcodec2 -lm
LD_LIBRARY_PATH=build/src ./gen_datac0 datac0_clean.s16       10 0   0    1  2
LD_LIBRARY_PATH=build/src ./gen_datac0 datac0_foff.s16        10 45  0    1  2
LD_LIBRARY_PATH=build/src ./gen_datac0 datac0_noise.s16       20 0   6000 7  2
LD_LIBRARY_PATH=build/src ./gen_datac0 datac0_noise_foff.s16  20 35  3000 11 2
```

## Burst-mode vectors (`datac*_burst*.s16`)

Generated by codec2 1.2.0 (git `310777b`)'s **own standard CLI tool**
`freedv_data_raw_tx` — the burst framing (`[silence][preamble][packet][postamble]
[2 frames of silence]` per burst) that the FreeDV tools and FreeDATA exchange
(`freedv_set_frames_per_burst(1)`). The burst RX tests
(`tests/.../Ofdm/DatacBurstTests.cs`) decode these with
`DatacReceiver(mode, packetsPerBurst: 1)`.

Payloads are deterministic: packet `p`, byte `i` = `(p·31 + i·17 + 5) & 0xff`
(the same formula the round-trip tests use). Every vector was verified
**byte-exactly** decodable by codec2's own `freedv_data_raw_rx` before check-in
(datac0 5/5, datac0-ch 5/5, datac3 2/2, datac4 2/2, datac4-ch 2/2,
datac13 2/2, datac13-ch 2/2, datac14 5/5, datac14-ch 5/5).

The datac4/13/14 vectors are the Phase-2 RX oracle for the narrow modes — the
ones whose receive path runs the `filtP200S400` RX band-pass filter and the
shortened LDPC codes. Their `_ch` variants carry a +22 Hz offset with AWGN at
SNR3k ≈ 5.4–5.8 dB (`SNRAv` reported by codec2's RX: datac4 4.9, datac13 6.4,
datac14 2.3 dB).

| file | mode | bursts × packets | channel | sha256 (first 12) |

| `datac0_burst_ch.s16`  | datac0  | 5 × 1 | `ch --freq 22 --No -20` (+22 Hz, SNR3k ≈ 4.6 dB) | `e7abcf8724f2` |
| `datac3_burst.s16`     | datac3  | 2 × 1 | clean | `e87778c72b57` |

### Regenerating

```sh
cd <codec2-1.2.0 checkout, git 310777b, built in build/>
python3 -c "
def gen(path, packets, nbytes):
    with open(path, 'wb') as f:
        for p in range(packets):
            f.write(bytes(((p*31 + i*17 + 5) & 0xff) for i in range(nbytes)))
gen('datac0_burst_in.bin', 5, 14)
gen('datac3_burst_in.bin', 2, 126)
gen('datac4_burst_in.bin', 2, 54)
gen('datac13_burst_in.bin', 2, 14)
gen('datac14_burst_in.bin', 5, 3)
"
export LD_LIBRARY_PATH=build/src
for m in datac0 datac3 datac4 datac13 datac14; do
  ./build/src/freedv_data_raw_tx $m ${m}_burst_in.bin ${m}_burst.s16
done
for m in datac0 datac4 datac13 datac14; do
  ./build/src/ch ${m}_burst.s16 ${m}_burst_ch.s16 --freq 22 --No -20
done
```
